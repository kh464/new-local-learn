# 前端界面中文化设计

## 目标

将当前 Vue 前端中用户直接可见的英文文案改为中文，并在不修改后端接口结构与返回字段的前提下，把结果页中用于展示的英文状态、标签、空值提示、执行模式等统一映射为中文。

本次改动只涉及前端显示层，不改后端接口、不改任务状态枚举定义、不改分析结果对象字段名，也不改后端生成的 Markdown/HTML/PDF 文件内容。

## 范围

包含：

- 页面级固定文案中文化
  - 顶部导航、页面标题、表单提示、按钮文案、空态提示、错误态提示、任务详情页说明、管理页说明
- 组件级固定文案中文化
  - 结果卡片标题、副标题、下载按钮、时间线标题、错误组件标题、状态卡片说明
- 结果显示层映射中文化
  - `running`、`failed`、`succeeded`、`queued`、`cancelled` 等状态
  - `planned`、`deterministic`、`llm`、`fallback` 等执行模式和执行节点状态
  - `none`、`unknown`、`not detected`、`yes`、`no` 等通用显示值
- 页面 `<title>` 和 HTML 语言属性同步调整为中文语境
- 现有前端测试按新的中文文案更新

不包含：

- 后端接口返回值结构调整
- 后端分析内容翻译
- 数据库存储文案迁移
- 完整国际化框架接入

## 方案选择

### 方案一：逐组件直接改字面量

优点：

- 实现速度快

缺点：

- 文案分散，后续维护成本高
- 状态值和通用占位词容易出现翻译不一致
- 结果页会混杂大量内联判断

### 方案二：增加前端文案与映射层

优点：

- 固定文案和动态映射统一管理
- 状态、执行模式、空值提示可以复用
- 后续继续补中文或扩展双语时成本更低

缺点：

- 初次改造会多一个前端工具文件

推荐方案：方案二。

## 设计

### 1. 文案组织

新增一个前端显示文案模块，集中提供：

- 页面与组件的固定中文文案常量
- 状态值到中文的映射函数
- 执行模式、节点状态、布尔值、空值占位的映射函数

组件内部不再直接写散落的英文默认值，统一调用映射层。

### 2. 固定文案改造

需要覆盖的主要文件：

- `web/index.html`
- `web/src/App.vue`
- `web/src/pages/HomePage.vue`
- `web/src/pages/AdminPage.vue`
- `web/src/pages/TaskDetailPage.vue`
- `web/src/components/RepositorySubmitForm.vue`
- `web/src/components/TaskStatusCard.vue`
- `web/src/components/TaskEventTimeline.vue`
- `web/src/components/TaskErrorState.vue`
- `web/src/components/AnalysisResultView.vue`
- 如有其它可见英文组件，也一并纳入

中文化后要求：

- 导航、按钮、输入框占位、说明文字全部为中文
- 错误、加载、空态、待处理提示全部为中文
- 不保留“中英混写”的界面，除非是代码、URL、字段值、仓库名等原始数据

### 3. 结果页动态映射

结果页中动态数据分两类处理：

- 原样保留：
  - 仓库名
  - GitHub URL
  - 文件路径
  - 路由路径
  - 组件名、变量名、资源名等分析对象标识
- 前端映射为中文：
  - 状态值
  - 执行模式
  - 空值默认文案
  - 卡片标题和字段标签

例如：

- `unknown` 显示为 `未知`
- `not detected` 显示为 `未检测到`
- `none` 显示为 `无`
- `yes` / `no` 显示为 `是` / `否`
- `running` 显示为 `执行中`
- `failed` 显示为 `失败`
- `succeeded` 显示为 `成功`
- `queued` 显示为 `排队中`
- `planned` 显示为 `计划中`
- `deterministic` 显示为 `确定性执行`
- `llm` 显示为 `大模型生成`
- `fallback` 显示为 `回退`

### 4. 页面行为边界

本次只改显示文本，不改变已有交互流程：

- 提交任务逻辑不变
- 轮询与流式事件逻辑不变
- 下载接口不变
- 管理页筛选逻辑不变
- 测试仍然以现有组件行为为主，只更新断言文本

### 5. 测试策略

优先更新和补充现有前端测试：

- 页面级测试
  - 首页是否显示中文标题和按钮
  - 管理页是否显示中文区块和状态标签
  - 任务详情页是否显示中文错误、加载、操作按钮
- 组件级测试
  - 提交表单文案是否为中文
  - 结果页卡片标题、下载按钮、状态映射是否为中文
  - 时间线和错误组件是否显示中文

测试重点是：

- 固定文案替换完整
- 动态状态映射正确
- 原始业务数据未被误翻译

## 风险与处理

### 风险一：只改了一部分页面，导致中英文混杂

处理：

- 通过集中扫描组件英文文案定位文件
- 更新相关前端测试，避免漏改

### 风险二：把后端原始数据误翻译，影响辨识

处理：

- 只翻译标签和状态值
- 仓库名、URL、路径、组件名、接口路径等保留原始值

### 风险三：多个组件内各自写映射，导致翻译不一致

处理：

- 统一通过前端映射模块输出中文

## 验收标准

- 前端主流程页面不再出现面向用户的英文固定文案
- 任务状态、执行状态、空值占位在显示层为中文
- 结果页卡片标题、副标题、下载按钮全部为中文
- 现有前端相关测试通过
- 不修改后端接口和数据结构
