# 中文报告与阶段时间线设计

## 背景

当前任务详情页会把 SSE 事件逐条平铺，页面会被大量 `5% / 20% / 35%` 之类的阶段事件刷满，不适合作为主时间线展示。同时，报告生成链路和可选 LLM 教程增强仍以英文为主，和当前中文界面目标不一致。

## 目标

- 任务时间线改为固定阶段视图，主界面只展示阶段进展、当前阶段和最新错误/消息。
- 任务详情与分析结果相关前端文案改为正常中文，修复当前关键页面中的乱码文案。
- Markdown 报告、内置教程生成内容默认输出中文。
- LLM 教程增强增加硬性中文约束，并在返回非中文内容时触发回退，避免最终报告混入英文段落。

## 设计

### 前端时间线

- `TaskEventTimeline` 不再按事件数组逐条渲染。
- 使用固定阶段序列：拉取代码、扫描目录、识别栈、分析后端、分析前端、生成文档、完成。
- 组件根据当前 `TaskStatus.stage` 和 `TaskStatus.state` 计算每个阶段的状态：
  - 已完成
  - 进行中
  - 失败
  - 未开始
- 顶部显示整体进度条和当前阶段。
- 底部仅显示最新一条错误或消息，不再默认打印完整事件列表。

### 中文文案

- 修复 `TaskDetailPage`、`TaskStatusCard`、`TaskEventTimeline`、`AnalysisResultView` 的中文展示。
- 统一由 `web/src/presentation/copy.ts` 提供阶段、状态、执行模式中文映射。

### 中文报告

- `TutorComposer` 改为输出中文的新手学习指引。
- `MarkdownCompiler` 改为输出中文标题、章节名和字段标签。

### LLM 中文约束

- `TutorialLLMEnhancer` 的系统提示词改为中文，并显式要求“所有字段值必须使用简体中文”。
- 用户提示词前增加中文强约束前缀。
- 对 LLM 返回结果做最小中文校验；若关键字段缺少中文字符则抛错，由任务流水线沿用现有 fallback 机制回退到 deterministic 中文教程。

## 风险与控制

- 前端现有测试中有乱码断言，必须同步更新，否则会出现实现正确但测试失败。
- LLM 中文校验不能过度依赖文件名/框架名，因为这些值天然可能是英文；因此只校验解释性字段。
- 时间线不再平铺所有事件后，调试信息减少，因此保留最新错误/消息作为最小诊断上下文。
